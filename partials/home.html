Why authz?

ngHide and ngShow are great for conditionally displaying HTML and simple logic in contorllers/services/etc. can further protect application resources. But it's annoying to write essentially the same code in every new application. angular-authz lets you reuse the same authorization module accross most applications.

Alternatives

Other AngularJS modules deal with role based access control. angular-authz supports Apache Shiro style permissions. Choosing role or permission based access control is a server side issue, but using angular-authz allows you to leverage role based systems with a seamless upgrade to permission based systems.

Extensiblity

angular-authz is a collection of angular constructs (directives, providers, etc) which out of the box supports most applications needs. But these constructs are exensible allowing developers to add custom permission types among other things.
